SafeChem Lab

1. Contexte et problème

Question de départ: comment rendre la chimie plus active, plus mémorable et plus sûre à apprendre?

Ce qu'on observe

  • Apprentissage souvent théorique et peu manipulatoire.
  • Difficulté à relier protocole, sécurité et résultat final.
  • Peu de feedback sur les erreurs de raisonnement.

Conséquence pédagogique

  • Mémorisation fragile sur les mécanismes réactionnels.
  • Réflexes HSE appris mais peu entraînés en contexte.
  • Manque d'autonomie dans l'analyse d'un montage.

2. Solution produit: SafeChem Lab

Parcours clair

Chaque niveau propose un objectif de synthèse, des cartes à sélectionner et des pièges à éviter.

Mécanique active

Drag & drop en 3 phases: substances, méthode, pictogrammes HSE.

Feedback utile

Résumé des erreurs avec explication de chaque piège utilisé.

Motivation

Score en étoiles (temps + précision) et progression visible niveau par niveau.

3. Démo d'un défi type

Étape A

Lecture de mission: objectif réactionnel et contraintes.

Étape B

Sélection de substances dans les slots dédiés.

Étape C

Choix de la méthode expérimentale adaptée.

Étape D

Validation des cartes HSE puis résultat final.

Exemples actuels: estérification de Fischer et production d'acide nitrique avec cartes pièges contextualisées.

4. Architecture technique

Côté Unity

  • UI générée au runtime pour refléter les données JSON (niveaux, cartes, docs).
  • Scène Home: pager 3 écrans + header/footer fixes + menu settings.
  • Scène InGame: contrôleur runtime avec phases, drag & drop et score.
  • Profil local persistant: UUID, pseudo, progression, étoiles.

Côté data et backend

  • Contenus pilotés par JSON: niveaux, docs, substances, méthodes, HSE.
  • Texture maps / atlas pour accélérer la production d'assets.
  • API REST Python pour synchroniser création joueur et résultats.
  • Stack backend: FastAPI + PostgreSQL + Adminer sous Docker.

5. Pourquoi le modèle data-driven est clé

  • Ajouter un niveau revient à ajouter un JSON dans le dossier Levels.
  • Le JSON décrit: cartes affichées, cartes attendues, pièges, messages d'erreur, consignes.
  • Les fiches docs, substances, méthodes et HSE suivent la même logique.
  • Le gameplay est extensible sans refactor lourd du code C#.

Résultat: gain de vitesse pour itérer sur le contenu pédagogique, et meilleure autonomie côté design pédagogique.

6. Valeur pédagogique attendue

Engagement

Le format mission + score augmente l'attention sur la durée.

Compréhension

Les erreurs expliquées permettent un apprentissage par correction active.

Sécurité

Les choix HSE sont entraînés en contexte, pas isolés sur fiche théorique.

Mesure

Temps, erreurs, étoiles: indicateurs simples pour suivre la progression.

7. Business model

Offres envisagées

  • Licence établissement (lycées, CFA, écoles techniques).
  • Version entreprise pour onboarding sécurité/process.
  • Pack contenu spécifique métier (chimie fine, procédés, HSE).

Monétisation

  • Abonnement annuel par groupe d'apprenants.
  • Option de personnalisation des niveaux et scénarios.
  • Module analytics pour formateurs / responsables pédagogiques.

8. Marché et différenciation

  • Besoin fort en formation immersive et professionnalisante.
  • Différenciation: combinaison chimie + gameplay + sécurité opérationnelle.
  • Positionnement pratique: pas juste "quiz", mais raisonnement de montage complet.
  • Potentiel d'extension vers d'autres domaines techniques à risques.

Le projet est pensé "contenu d'abord": la valeur augmente à mesure que le catalogue de défis grandit.

9. Roadmap d'exécution

  1. Finaliser le prototype jouable end-to-end (stabilité mobile + contenus clés).
  2. Lancer un pilote pédagogique sur un petit groupe cible.
  3. Mesurer: taux de complétion, erreurs récurrentes, progression entre tentatives.
  4. Industrialiser le pipeline contenu pour créer rapidement de nouveaux niveaux.
  5. Préparer une version v1 orientée déploiement établissement.

10. Conclusion et appel à action

SafeChem Lab rend la chimie plus concrète, plus engageante et plus sûre à apprendre. La base technique est en place, le modèle contenu est extensible, et la prochaine étape est la validation terrain avec des apprenants réels.
